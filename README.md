# Logoust Assistant

Smart-помощник логопедического кабинета Logoust. Снижает рутину специалиста и даёт удобный канал напоминаний и информации для клиентов.

## Ключевые возможности

- **Онбординг специалиста через invite-ссылку.** Администратор создаёт одноразовое приглашение командой `make create_invite`, передаёт специалисту Telegram-ссылку `https://t.me/<bot>?start=<token>`. Подключиться можно двумя способами: перейти по ссылке (бот получит `/start <token>`) либо вставить в чат с ботом голый код или ссылку целиком — бот сам извлечёт токен. По первому валидному токену бот привязывает Telegram-аккаунт к записи и отправляет приветствие. Повторное подключение безопасно — приветствие не дублируется.
- **Управление списком клиентов.** Картотека клиентов специалиста прямо в Telegram (reply-кнопка «👶 Клиенты» → inline-меню). Добавление через пошаговый визард (имя ребёнка, контакт родителя, телефон, Telegram), редактирование любого поля, архивация и возврат без потери данных, отдельные списки активных и архивных. Телефон нормализуется к `+7XXXXXXXXXX`. Каждый специалист видит только своих клиентов.
- **Приглашение клиента в бота.** Из карточки специалист получает пересылаемую ссылку-приглашение; клиент в один тап привязывает свой Telegram к карточке — канал связи для будущих уведомлений. Один аккаунт можно привязать к нескольким карточкам (удобно для тестов без второй учётки).
- **Записи клиентов на приём.** Из карточки клиента: дата (инлайн-календарь, прошлое заблокировано), время (сетка слотов или произвольный `ЧЧ:ММ`) и опциональный комментарий. Reply-кнопка «📅 Расписание» показывает будущие записи всех клиентов по дням и историю с пагинацией; в карточке клиента — его будущие записи и история. Запись можно перенести или удалить (с подтверждением, жёстко). После создания, переноса или отмены — если клиент привязан к боту — бот отдельным шагом спрашивает «Уведомить клиента?» с предпросмотром текста и по кнопке «Да» отправляет клиенту уведомление о событии. Время хранится в UTC и считается по таймзоне специалиста.
- **Регулярные записи.** При создании обычной записи бот спрашивает, сделать ли её регулярной — встреча начинается с выбранной даты и повторяется каждую неделю, пока её не остановят. Будущие повторы занимают слот во всех экранах (календарь, окна, расписание, карточка клиента) и помечены значком 🔁; прошедшие встречи становятся обычными записями в истории. Серию можно настроить или отменить, а отдельную дату — отменить или перенести.
- **Напоминания клиентам о записи.** Раз в день (по умолчанию 12:00 настенного времени специалиста) бот сам напоминает привязанному клиенту о записи на завтра и собирает подтверждение кнопками «✅ Подтверждаю» / «❌ Не смогу». Подтверждение видно специалисту (✅ в расписании и пометка на карточке), при отказе ему сразу приходит уведомление с переходом на запись. Можно выключить, время настраивается. Работает и для повторов регулярных серий.
- **Абонементы.** Учёт абонементов клиента на несколько встреч: кнопка на карточке клиента создаёт абонемент (с дефолтным числом встреч) или ведёт на его карточку с остатком. На карточке — сколько куплено и осталось, ручное списание встречи, продление на N встреч и закрытие с подтверждением. Отдельный раздел «🎫 Абонементы» в меню показывает список активных абонементов всех клиентов и историю закрытых. Не более одного активного абонемента на клиента; закрытые сохраняются. Дефолтное число встреч настраивается.
- **Настройки расписания.** Reply-кнопка «⚙️ Настройки»: таймзона (из списка российских зон), начало и конец рабочего дня, длина слота, рабочие дни недели (мульти-тоггл Пн–Вс), напоминания клиентам (вкл/выкл и время), дефолтное число встреч в абонементе. Параметры задают сетку времени при создании записей и набор рабочих дней для свободных окон.
- **Свободные окна.** Reply-кнопка «🪟 Окна»: read-only обзор свободного времени на ближайшие 5 рабочих дней, сгруппированный по дням. Свободное окно — слот сетки без записи; для сегодня прошедшие окна скрыты.

## Технологический стек

- **Python 3.13**, async-first.
- **aiogram 3** — Telegram-бот в режиме long-polling.
- **SQLAlchemy 2.0** (async) + **aiosqlite** — ORM и драйвер. По умолчанию SQLite.
- **Alembic** — миграции (sync, отдельный URL).
- **pydantic-settings** — конфиг из `.env`.
- **structlog** — структурированные логи.
- **pytest** + **pytest-asyncio** + **pytest-cov** — тесты, требование 100% покрытия.
- **Ruff** — форматтер и линтер; **ty** — type-checker.
- **uv** — менеджер зависимостей и виртуального окружения.

## Архитектура

Clean Architecture под `src/`:

- `domain/` — pure-Python сущности и протоколы репозиториев.
- `services/` — use-case-функции, зависят только от `domain/`.
- `infrastructure/` — SQLAlchemy ORM, async-сессии, реализации репозиториев.
- `bot/` — aiogram dispatcher, роутеры, хендлеры и каталог пользовательских текстов.
- `cli/` — административные точки входа (например, `create_invite`).

Развёрнутая документация в [`docs/`](./docs/README.md).

## Быстрый старт

1. Установить **Python 3.13** и **uv**.
2. Скопировать `.env.example` в `.env` и заполнить:
   - `TELEGRAM_BOT_TOKEN` — токен бота из BotFather (обязательная).
   - `TELEGRAM_BOT_USERNAME` — username бота без `@` (обязательная). Нужен CLI для печати deep-link.
   - `DATABASE_URL` — по умолчанию `sqlite+aiosqlite:///./logoust.db`.
3. `make init` — установить зависимости и pre-commit-хуки.
4. `make create_invite` — создать приглашение и получить deep-link для специалиста.
5. `make run` — применить миграции и запустить бота.

## Команды разработки

- `make init` — установить зависимости и pre-commit-хуки.
- `make run` — `alembic upgrade head` + запуск бота в long-polling.
- `make run-reload` — то же, но с авто-перезапуском при изменениях в `src/` и `alembic/` (для разработки).
- `make create_invite` — создать приглашение специалиста и напечатать deep-link.
- `make check` — формат + линт + type-check + тесты (требуется до commit).
- `make format` / `make lint` / `make type-check` / `make test` — отдельные шаги.
- `make upgrade` — обновить Python, lockfile, pre-commit.
- `make clean` — удалить кэши и `__pycache__`.
