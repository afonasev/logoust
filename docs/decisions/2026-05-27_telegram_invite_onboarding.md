# 2026-05-27 · Онбординг специалиста через Telegram invite-ссылку

## Контекст

Первое подключение специалиста к ассистенту. Нужно завести его в базе, привязать к Telegram-аккаунту и отправить приветствие, чтобы дальше иметь канал для напоминаний и заметок.

## Решения

### Deep-link с одноразовым токеном (Flow B)

Администратор создаёт запись с `invite_token`, передаёт специалисту ссылку `https://t.me/<bot>?start=<token>`. Специалист переходит → Telegram доставляет боту `/start <token>` → бот сопоставляет токен с записью.

- **Почему не «бот пишет первым по username»:** Telegram-бот не может инициировать диалог. До первого `/start` от пользователя у бота нет `chat_id`, чтобы кому-то писать. Это политика Telegram, обходов нет.
- **Почему не ручной ввод `chat_id`:** требует поход к BotFather или сторонним инструментам, ошибки наборщика, заметная UX-просадка для специалиста.
- **Почему deep-link:** один клик у специалиста, нулевая нагрузка на администратора (передать ссылку — и всё), идемпотентность даётся бесплатно через `welcomed_at`.
- **Цена:** при «утечке» ссылки до её использования первый, кто перейдёт, привяжет аккаунт; защита от этого вынесена в отдельное изменение (revocation).

### aiogram 3

- **Почему:** актуальный async-фреймворк для Telegram-ботов на Python, типизированные апдейты, родной router/FSM, активная поддержка.
- **Альтернативы:** `python-telegram-bot` (sync-первый, медленнее догоняет API), голый HTTP-клиент (придётся писать диспетчер самим — YAGNI).
- **Цена:** ещё одна нетривиальная зависимость с собственным релиз-циклом; держим pin `>=3,<4`.

### TOML для каталога текстов

Все пользовательские тексты в `src/bot/messages.toml`, грузятся `tomllib` (stdlib) в frozen-dataclass.

- **Почему TOML:** multi-line strings из коробки, без YAML-неоднозначностей с отступами и неявными типами, без новой зависимости.
- **Почему не YAML:** ловушки с нечётным форматированием, требует `PyYAML`.
- **Почему не Python-модуль:** редактирование текстов не должно требовать понимания экранирования; синтаксическая ошибка в `.py` уронит весь бот, в TOML — только парсинг.
- **Fail-fast:** отсутствие обязательного ключа на старте → `RuntimeError`, не «молча шлём None».
- **Цена:** ещё один формат файла в репозитории; считаю приемлемой ценой за стабильность редактирования.

### Async-only через SQLAlchemy

aiogram async, SQLAlchemy `AsyncSession`, `aiosqlite`. CLI оборачивает корневую корутину в `asyncio.run`.

- **Почему не sync-репозитория + `asyncio.to_thread`:** иначе через сервисный слой пришлось бы таскать два разных стиля сессий — плата за смешение выше, чем за «всё async».
- **Цена:** Alembic остаётся sync (для миграций это и проще, и стандартнее), приходится конвертировать URL.

### SQLite по умолчанию

- **Почему:** одна офисная инсталляция, минимум административных затрат, файл на диске.
- **Цена:** придётся аккуратно отнестись к одновременному доступу при росте; для текущего объёма (десятки специалистов) это не проблема.
- **План:** при росте — миграция на Postgres через тот же `DATABASE_URL`, ORM-код не меняется.
