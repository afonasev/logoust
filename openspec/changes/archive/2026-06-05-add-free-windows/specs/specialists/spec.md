## MODIFIED Requirements

### Requirement: Настройки расписания специалиста

Каждый специалист **SHALL** иметь настройки расписания: таймзону (`timezone`, IANA-строка),
начало рабочего дня (`day_start`), конец рабочего дня (`day_end`), длину слота в минутах
(`slot_minutes`) и рабочие дни недели (`working_days`). Эти настройки **SHALL** использоваться
для генерации сетки времени при создании и переносе записей, для трактовки настенного времени
относительно таймзоны специалиста и для расчёта свободных окон. `working_days` хранится как
канонически отсортированная строка индексов дней недели (понедельник = 0 … воскресенье = 6).
Существующим специалистам при миграции **SHALL** проставляться значения по умолчанию:
`timezone = Asia/Yekaterinburg`, `day_start = 09:00`, `day_end = 20:00`, `slot_minutes = 60`,
`working_days = понедельник–пятница`.

#### Scenario: Дефолтные настройки у существующего специалиста

- **WHEN** применяется миграция, добавляющая поля настроек расписания
- **THEN** у каждой существующей записи `specialists` проставляются `timezone = Asia/Yekaterinburg`, `day_start = 09:00`, `day_end = 20:00`, `slot_minutes = 60`, `working_days = понедельник–пятница`

#### Scenario: Генерация сетки слотов из настроек

- **WHEN** специалист с `day_start = 14:00`, `day_end = 19:40`, `slot_minutes = 50` открывает выбор времени
- **THEN** сетка слотов начинается с `day_start` и шагает по `slot_minutes`, пока старт слота меньше `day_end` (`14:00, 14:50, 15:40, 16:30, 17:20, 18:10, 19:00`)

### Requirement: Изменение настроек расписания в боте

Бот **SHALL** предоставлять специалисту в главном меню раздел настроек, где он может изменить
таймзону (выбором из списка российских таймзон), начало рабочего дня, конец рабочего дня,
длину слота и рабочие дни недели. При сохранении новое значение **SHALL** немедленно
применяться к последующей генерации сетки, трактовке времени и расчёту свободных окон.
Рабочие дни недели **SHALL** редактироваться мульти-тогглом (Пн–Вс): нажатие на день
инвертирует его и сразу сохраняется, без отдельного шага подтверждения.

#### Scenario: Смена таймзоны

- **WHEN** специалист открывает настройки и выбирает другую таймзону из списка
- **THEN** в его записи `specialists` сохраняется выбранная IANA-таймзона
- **AND** последующие создание и показ записей считаются в новой таймзоне

#### Scenario: Смена параметров рабочего дня

- **WHEN** специалист изменяет начало дня, конец дня или длину слота на корректные значения
- **THEN** новое значение сохраняется и применяется к следующей сетке слотов

#### Scenario: Переключение рабочего дня недели

- **WHEN** специалист в настройках нажимает на день недели в тоггле рабочих дней
- **THEN** этот день инвертируется (рабочий ↔ нерабочий), новый набор сразу сохраняется в `specialists`
- **AND** расчёт свободных окон начинает учитывать обновлённый набор рабочих дней
