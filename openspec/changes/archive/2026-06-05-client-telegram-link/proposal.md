## Why

Сейчас у клиента в картотеке есть только `contact_telegram` (username-строка), а по
username Telegram писать боту **нельзя** — для отправки сообщения нужен `chat_id`,
который появляется только когда пользователь сам стартует бота. Без захвата `chat_id`
невозможны никакие уведомления клиенту (напоминания о записи и т.п.). Этот change
заводит канал связи с клиентом: специалист приглашает клиента в бота, клиент
переходит по deep-link, бот привязывает его `chat_id`. Это фундамент для последующего
change `appointment-notify`.

## What Changes

- Специалист может из карточки клиента сгенерировать приглашение в бота — бот выдаёт
  готовое **пересылаемое** сообщение с deep-link `t.me/<bot>?start=cli_<token>` и
  короткой подсказкой клиенту нажать «Запустить».
- Клиент, перейдя по ссылке (один тап по системной кнопке «Запустить», без ввода
  команд), привязывает свой Telegram к карточке: бот сохраняет `telegram_chat_id` и
  `linked_at`.
- Повторный переход по ссылке — **идемпотентный rebind**: `chat_id` перезаписывается
  (удобно для тестов и смены устройства/аккаунта клиентом).
- Один аккаунт Telegram может одновременно быть и специалистом, и клиентом (для
  тестов без второй учётки): роутинг оператора всегда идёт через `specialists`,
  клиентский онбординг — только по явному префиксу `cli_`.
- На карточке клиента кнопка отражает состояние: «Пригласить в бота» → после привязки
  «✅ Привязан (переотправить приглашение)».
- Расширяется таблица `clients`: `telegram_chat_id`, `invite_token`, `linked_at`.

## Capabilities

### New Capabilities

(нет — поведение расширяет существующую картотеку клиентов)

### Modified Capabilities

- `clients`: добавляются требования о приглашении клиента в бота, генерации deep-link,
  привязке `telegram_chat_id` через онбординг (идемпотентный rebind), сосуществовании
  одного Telegram-аккаунта в роли специалиста и клиента, и отображении состояния
  привязки в карточке.

## Impact

- **Данные/миграция**: новые колонки в `clients` (`telegram_chat_id BIGINT NULL`,
  `invite_token TEXT NULL UNIQUE`, `linked_at TIMESTAMP NULL`).
- **domain**: `Client` (+поля), `ClientsRepo` (+`find_by_invite_token`, `link_telegram`,
  `set_invite_token`).
- **services**: новые use-case'ы генерации приглашения и привязки клиента по токену.
- **infrastructure**: `SqlAlchemyClientsRepo` + ORM-модель `clients`.
- **bot**: карточка клиента (кнопка приглашения), `start.py` (роутинг `cli_`-токена и
  fallback-обработчик вставленного текста/ссылки), новые тексты в `messages.toml`.
- **Документация**: `docs/database.md`, `docs/bot.md`, `docs/features.md`,
  `openspec/specs/clients/spec.md`, `README.md`.
- Не затрагивает онбординг специалистов и записи (записи — следующий change).
