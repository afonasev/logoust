## Why

Подтверждение записей — рутина, которую специалист сейчас делает вручную (звонит/пишет
накануне). Канал связи с привязанным клиентом уже есть (`clients.telegram_chat_id`), сборка
записей на день — тоже (`list_specialist_day`). Нужно, чтобы бот сам накануне напоминал
клиенту о встрече и собирал подтверждение, а специалист видел итог одним взглядом и узнавал
об отказе сразу, без переклички.

## What Changes

- Появляется **автоматическое напоминание клиенту о записи на завтра**: раз в день
  (по умолчанию **12:00** настенного времени специалиста) бот сам шлёт каждому привязанному
  клиенту, у которого есть запись на завтра, сообщение с датой/временем и двумя кнопками —
  **«✅ Подтверждаю»** и **«❌ Не смогу прийти»**. Непривязанным клиентам (`telegram_chat_id
  IS NULL`) ничего не шлётся.
- Ответ клиента сохраняется как **статус напоминания** (`pending` / `confirmed` /
  `declined`). Клиент может **изменить ответ в любой момент** — статус перезаписывается.
- **Подтверждение** показывается специалисту: ✅ напротив записи в дневном экране расписания
  и пометка «подтверждена» на карточке записи.
- **Отказ** отправляет специалисту сообщение с кнопкой **«→ Открыть запись»**; автоматической
  отмены нет — специалист сам решает, что делать (перенести/отменить существующими сценариями).
  Уведомление специалисту шлётся при каждом переходе ответа `* → declined`.
- Напоминание можно **выключить**, время отправки — **настраиваемое** (дефолт 12:00,
  по умолчанию включено) — по образцу настроек утренней сводки.
- Используется **общий планировщик-петля** (минутный тик рядом с polling): в него
  регистрируются due-джобы; материализует петлю тот из changes (`daily-digest` /
  `appointment-reminder`), что мёрджится первым.
- Только батч в заданное время: запись на завтра, **созданная после прохода**, авто-напоминания
  не получает (специалист может уведомить вручную через `appointment-notify`).

## Capabilities

### New Capabilities

- `appointment-reminder`: автоматическое напоминание привязанному клиенту о записи на завтра
  с подтверждением — расписание прохода (настенное время в tz, антидубль, догон после
  простоя), правила содержимого (только привязанные клиенты, записи на завтра, включая
  виртуальные повторы серий), журнал статусов подтверждения (привязка к occurrence по
  `(client_id, starts_at)`, работает и для виртуального будущего серий), реакция на ответ
  клиента (перезаписываемый статус, уведомление специалиста об отказе), настройки специалиста
  (вкл/выкл, время).

### Modified Capabilities

- `appointments`: запись отображает **статус подтверждения** клиента — ✅ в дневном экране
  расписания и пометка на карточке записи; источник статуса — журнал напоминаний.

## Impact

- **Схема БД**: новая таблица `appointment_reminders` (журнал: `specialist_id`, `client_id`,
  `starts_at`, `series_id`/`origin_date`, `status`, `sent_at`, `responded_at`,
  `UNIQUE(specialist_id, client_id, starts_at)`) + новые колонки на `specialists`
  (`reminder_enabled`, `reminder_time`, `reminder_last_run_on`) + миграция alembic.
- **domain**: сущность напоминания и протокол репозитория; чистая функция «кому пора»
  (`is_reminder_due`); новые поля `Specialist`.
- **services**: use-case прохода (сбор записей на завтра → вставка журнала → отдача текста для
  отправки), use-case обработки ответа (перезапись статуса, сигнал об отказе), чтение статусов
  для дневного экрана/карточки.
- **bot**: фоновая джоба в общем minute-loop (`src/__main__.py`); колбэк `appt:cfm:` (ответ
  клиента); уведомление специалисту об отказе с кнопкой на карточку; ✅ в дневном экране и на
  карточке записи (`src/bot/handlers/schedule.py`); тумблер/время в настройках
  (`src/bot/handlers/settings.py`). Отправка — в слое `bot/`, сервисы без aiogram.
- **Тексты**: новые ключи в `messages.toml` (текст напоминания клиенту, подписи кнопок,
  уведомление об отказе специалисту, пометки статуса, пункты настроек).
- **Логи**: `appointment.reminder_sent` / `appointment.reminder_confirmed` /
  `appointment.reminder_declined` / `appointment.reminder_failed`.
- **Документация**: `docs/database.md`, `docs/bot.md`, `docs/features.md`,
  `docs/architecture.md` (общий планировщик-петля), `openspec/specs/`.
