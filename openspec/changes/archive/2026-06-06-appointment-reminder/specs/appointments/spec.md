## ADDED Requirements

### Requirement: Отображение статуса подтверждения записи

Запись **SHALL** отображать специалисту статус подтверждения клиента, полученный через
напоминание (см. capability `appointment-reminder`). Источник статуса — журнал напоминаний,
сопоставляемый с записью по клиенту и времени начала. Отображение **SHALL** одинаково работать
для разовых записей и виртуальных повторов серий.

В дневном экране расписания запись с подтверждённым статусом **SHALL** помечаться визуальным
индикатором подтверждения. На карточке записи с подтверждённым статусом **SHALL** показываться
пометка о подтверждении. Записи без подтверждения (нет напоминания, статус `pending` или
`declined`) индикатор подтверждения **MUST NOT** получать. Все формулировки **MUST** браться
из `src/bot/messages.toml`.

#### Scenario: Подтверждённая запись в дневном экране

- **WHEN** специалист открывает дневной экран расписания и у записи статус напоминания `confirmed`
- **THEN** запись помечена индикатором подтверждения

#### Scenario: Пометка подтверждения на карточке записи

- **WHEN** специалист открывает карточку записи со статусом напоминания `confirmed`
- **THEN** на карточке показана пометка о подтверждении

#### Scenario: Неподтверждённая запись без индикатора

- **WHEN** у записи нет напоминания либо статус `pending`/`declined`
- **THEN** запись не помечается индикатором подтверждения
