## ADDED Requirements

### Requirement: Каталог настраиваемых клиентских шаблонов

Система **SHALL** хранить каталог настраиваемых текстов как набор именованных **клиентских**
шаблонов (`template_key`). В каталог **SHALL** входить только сообщения, отправляемые клиенту;
тексты, которые видит сам специалист (меню, кнопки, подсказки ввода), в каталог **MUST NOT**
входить. Для каждого `template_key` каталог **SHALL** задавать набор разрешённых плейсхолдеров
(whitelist) и набор обязательных плейсхолдеров (подмножество разрешённых). Whitelist клиентских
шаблонов **MUST NOT** содержать плейсхолдеры с персональными данными (имя ребёнка, имя контакта)
— разрешены только обезличенные значения (например, дата, время, ссылка-приглашение). Каждый
`template_key` каталога **SHALL** иметь дефолтный текст в `messages.toml`.

#### Scenario: Только клиентские тексты в каталоге

- **WHEN** запрашивается список настраиваемых шаблонов
- **THEN** в нём присутствуют клиентские сообщения (напоминание о записи, уведомления о
  создании/переносе/отмене, просьба продлить абонемент, приглашение, подтверждение привязки)
- **AND** в нём отсутствуют тексты интерфейса специалиста (пункты меню, подписи кнопок, подсказки)

#### Scenario: Плейсхолдеры с именами недоступны

- **WHEN** для любого клиентского шаблона запрашивается набор разрешённых плейсхолдеров
- **THEN** в нём нет плейсхолдера с именем ребёнка или контакта
- **AND** присутствуют только обезличенные плейсхолдеры (дата/время/ссылка), предусмотренные для
  этого шаблона

#### Scenario: У каждого шаблона есть дефолт

- **WHEN** для `template_key` из каталога отсутствует переопределение специалиста
- **THEN** при разрешении текста возвращается дефолт этого ключа из `messages.toml`

### Requirement: Переопределение и сброс текста шаблона

Каждый специалист **SHALL** иметь возможность переопределить текст любого клиентского шаблона из
каталога своим текстом и сбросить его обратно к дефолту. Переопределение **SHALL** храниться
отдельно по специалисту и ключу (одно переопределение на пару `specialist_id` + `template_key`);
сохранение нового текста для того же ключа **SHALL** заменять предыдущее. Сброс **SHALL** удалять
переопределение, после чего шаблон снова разрешается в дефолт. Дефолтные тексты в `messages.toml`
**MUST NOT** изменяться переопределением.

#### Scenario: Переопределение клиентского текста

- **WHEN** специалист сохраняет валидный текст для шаблона `appt_reminder`
- **THEN** для этого специалиста сохраняется переопределение `appt_reminder`
- **AND** при разрешении `appt_reminder` для этого специалиста возвращается его текст, а не дефолт

#### Scenario: Замена существующего переопределения

- **WHEN** у специалиста уже есть переопределение шаблона, и он сохраняет для того же шаблона
  другой валидный текст
- **THEN** прежнее переопределение заменяется новым (одно переопределение на ключ)

#### Scenario: Сброс к дефолту

- **WHEN** специалист сбрасывает переопределённый шаблон
- **THEN** переопределение удаляется
- **AND** при разрешении этого шаблона снова возвращается дефолт из `messages.toml`

#### Scenario: Изоляция между специалистами

- **WHEN** один специалист переопределил шаблон
- **THEN** у другого специалиста тот же шаблон по-прежнему разрешается в дефолт

### Requirement: Строгая валидация плейсхолдеров при сохранении

Перед сохранением переопределения система **SHALL** проверять текст строго: каждый плейсхолдер
в тексте **MUST** входить в whitelist этого `template_key`, и каждый обязательный плейсхолдер
этого ключа **MUST** присутствовать в тексте. Текст с недопустимым плейсхолдером, без
обязательного плейсхолдера или пустой **MUST NOT** сохраняться; специалисту **SHALL** возвращаться
понятное сообщение с указанием доступных и обязательных плейсхолдеров. Валидный текст **SHALL**
сохраняться.

#### Scenario: Недопустимый плейсхолдер

- **WHEN** специалист сохраняет для `appt_reminder` текст с плейсхолдером вне whitelist (например, `{child}` или `{foo}`)
- **THEN** текст не сохраняется
- **AND** специалисту сообщается, какие плейсхолдеры допустимы для этого шаблона

#### Scenario: Пропущен обязательный плейсхолдер

- **WHEN** специалист сохраняет для `appt_reminder` текст без обязательного `{time}`
- **THEN** текст не сохраняется
- **AND** специалисту сообщается, что `{time}` обязателен

#### Scenario: Пустой текст

- **WHEN** специалист присылает пустой текст
- **THEN** текст не сохраняется
- **AND** специалисту сообщается, что текст не может быть пустым

#### Scenario: Валидный текст сохраняется

- **WHEN** специалист сохраняет текст со всеми обязательными и только разрешёнными плейсхолдерами
- **THEN** переопределение сохраняется

### Requirement: Разрешение текста клиентского сообщения

При формировании любого сообщения клиенту бот **SHALL** брать текст шаблона через разрешение
«переопределение специалиста, иначе дефолт» по `template_key`, а не читать `messages.toml`
напрямую. Подстановка конкретных значений (дата, время, ссылка) в разрешённый шаблон выполняется
местом отправки. Поскольку сохранённое переопределение прошло строгую валидацию, в нём
**SHALL** присутствовать ровно ожидаемые плейсхолдеры, и подстановка **SHALL NOT** приводить к
ошибке из-за лишних или неизвестных плейсхолдеров.

#### Scenario: Используется переопределение специалиста

- **WHEN** бот формирует клиентское сообщение для специалиста, у которого этот шаблон переопределён
- **THEN** в основу берётся текст специалиста с подставленными значениями

#### Scenario: Используется дефолт

- **WHEN** бот формирует клиентское сообщение для специалиста без переопределения этого шаблона
- **THEN** в основу берётся дефолтный текст из `messages.toml` с подставленными значениями

### Requirement: Раздел настроек «Тексты для клиента»

Бот **SHALL** предоставлять в меню настроек раздел со списком клиентских шаблонов. Для каждого
шаблона раздел **SHALL** давать действие правки и действие сброса к дефолту. При правке бот
**SHALL** показывать текущий текст шаблона и список доступных плейсхолдеров с пометкой
обязательных, после чего принимать новый текст и применять к нему строгую валидацию.

#### Scenario: Открытие раздела

- **WHEN** специалист открывает раздел «Тексты для клиента»
- **THEN** показывается список клиентских шаблонов с действиями правки и сброса для каждого

#### Scenario: Подсказка плейсхолдеров при правке

- **WHEN** специалист начинает правку шаблона
- **THEN** показывается текущий текст и перечень доступных плейсхолдеров с пометкой, какие обязательны

#### Scenario: Сброс из раздела настроек

- **WHEN** специалист нажимает сброс у переопределённого шаблона
- **THEN** переопределение удаляется и шаблон возвращается к дефолту
