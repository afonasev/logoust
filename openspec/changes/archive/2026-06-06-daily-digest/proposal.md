## Why

Специалист каждое утро вручную открывает расписание, чтобы понять план на день. Это рутина
и риск что-то упустить. Канал связи со специалистом уже есть (`specialists.telegram_chat_id`),
а сборка записей на день — тоже (`list_specialist_day`). Нужно один раз утром само́му боту
прислать специалисту готовый список встреч на сегодня.

## What Changes

- Появляется **утренняя сводка специалисту**: один раз в день бот сам присылает специалисту
  текстовое сообщение со списком встреч, назначенных на сегодня. Без кнопок и интерактива.
- Время по умолчанию **10:00** (настенное в таймзоне специалиста), его можно изменить в
  настройках. Уведомление можно **выключить**. По умолчанию для нового специалиста — **включено**.
- В настройках есть кнопка **«Отправить сейчас»** для проверки: присылает сводку немедленно,
  мимо расписания и не затрагивая антидубль (настоящая утренняя сводка остаётся в силе).
- Сводка **не отправляется в день без записей** (пустой или нерабочий день — бот молчит).
- В список включаются и виртуальные повторы активных серий — так же, как их показывает
  дневной экран расписания (`list_specialist_day(series=...)`).
- Впервые в проекте появляется **фоновая задача**: рядом с long-polling в `src/__main__.py`
  крутится asyncio-таск с минутным тиком, который решает, кому сегодня уже пора слать.
- Антидубль: на специалисте хранится дата последнего «решения за день»; повторный тик и
  рестарт процесса не приводят к повторной отправке.

## Capabilities

### New Capabilities

- `daily-digest`: автоматическая утренняя сводка специалисту со списком встреч на сегодня —
  расписание отправки (настенное время в tz, антидубль, догон после простоя), правила
  содержимого (пустой день не шлётся, серии включаются), а также настройки специалиста
  (вкл/выкл, время).

### Modified Capabilities

(нет — поведение не меняет существующие capability записей/настроек на уровне их требований)

## Impact

- **Схема БД**: новые колонки на `specialists` (`morning_notify_enabled`,
  `morning_notify_time`, `morning_notify_last_run_on`) + миграция alembic. Дефолты —
  и server-default миграции, и значения в домене `Specialist`.
- **Новый слой поведения**: фоновый asyncio-таск в `src/__main__.py`
  (`asyncio.gather(start_polling, digest_loop)`); падение таска не должно ронять polling.
- **domain**: чистая функция «кому пора» (`is_digest_due` или аналог) — тестируется без таймера;
  новые поля и методы репозитория специалистов (`list_digest_candidates`, `mark_digest_run`).
- **services**: use-case прохода по специалистам — сбор среза дня, рендер текста, пометка
  «обработан», логи `specialist.digest_sent` / `specialist.digest_skipped_empty`.
- **bot**: новый пункт в меню настроек (`src/bot/handlers/settings.py`) — тумблер вкл/выкл и
  ввод времени (через готовый `parse_hhmm`, FSM по образцу `day_start`). Отправка сводки —
  в слое `bot/` (`bot.send_message`), сервис остаётся без aiogram.
- **Тексты**: новые ключи в `messages.toml` (заголовок/строки сводки, пункты меню настроек).
- **Документация**: `docs/architecture.md` (фоновый таск + новая переменная конфигурации
  интервала, если появится), `docs/database.md` (колонки + миграция), `docs/bot.md`
  (настройки), `docs/features.md` (сценарий), `openspec/specs/daily-digest/spec.md`.
