## ADDED Requirements

### Requirement: Ручной немедленный запуск напоминаний

Специалист **SHALL** иметь в настройках кнопку ручного запуска, которая немедленно
выполняет проход напоминаний о записях на завтра — в обход настенного времени
`reminder_time`, дневного флага `reminder_last_run_on` и флага вкл/выкл (паритет с
ручной отправкой утренней сводки). Ручной запуск **MUST NOT** проставлять
`reminder_last_run_on` — боевой дневной проход за этот день выполняется как обычно.

Ручной запуск **SHALL** использовать тот же per-occurrence антидубль по журналу
`appointment_reminders` (UNIQUE `specialist_id, client_id, starts_at`), что и боевой
проход: по occurrence, по которой напоминание уже отправлялось, повторное напоминание
**MUST NOT** уходить. Поэтому реальные клиенты, которым уже слали сегодня, повторно
ничего не получают, а запись на завтра, добавленная после боевого прохода, получает
напоминание. Отправка, текст (шаблон `appt_reminder`), кнопки подтверждения/отказа и
журналирование в `audit_log` (SENT/FAILED) **SHALL** быть идентичны боевому проходу;
сбой доставки одному клиенту **MUST NOT** прерывать остальных. По завершении специалисту
**SHALL** показываться результат: сколько напоминаний отправлено либо что напоминать на
завтра некого.

#### Scenario: Ручной запуск отправляет напоминание по новой записи

- **WHEN** специалист нажимает кнопку ручного запуска, а на завтра есть запись привязанного клиента, по которой напоминание ещё не отправлялось
- **THEN** клиенту немедленно отправляется напоминание с датой/временем и кнопками подтверждения/отказа, отправка журналируется в `audit_log`
- **AND** `reminder_last_run_on` не проставляется, и боевой дневной проход за этот день выполняется как обычно

#### Scenario: Повторный ручной запуск не дублирует уже отправленное

- **WHEN** специалист нажимает кнопку ручного запуска повторно, а по записям на завтра напоминания уже были отправлены (есть строки журнала)
- **THEN** повторные напоминания этим клиентам не отправляются
- **AND** специалисту показывается, что новых отправок нет

#### Scenario: Ручной запуск работает при выключенных напоминаниях

- **WHEN** у специалиста напоминания выключены, и он нажимает кнопку ручного запуска
- **THEN** проход всё равно выполняется и напоминания по записям на завтра отправляются

#### Scenario: На завтра напоминать некого

- **WHEN** специалист нажимает кнопку ручного запуска, а на завтра нет записей привязанных клиентов без уже отправленного напоминания
- **THEN** ничего не отправляется и специалисту показывается подсказка, что напоминать некого
