## Context

Шаги ввода свободного текста существуют в двух местах:

- `settings.py::ask_value` — единый вход для 7 полей (`day_start`, `day_end`, `slot`,
  `reminder_time`, `digest_time`, `subscription_presets`, `deferred_time`). Сейчас он
  делает `edit_text(self._prompt(field))` без клавиатуры и без загрузки специалиста.
- `schedule.py::notify_custom` — шаг «Своё время» отложенного уведомления: `edit_text`
  с голым промптом, без клавиатуры. Это пост-action шаг (запись уже создана), часть
  notify-флоу с экраном выбора момента (`_notify_when_keyboard`: Сейчас / в HH:MM / Своё время).

Существующие переиспользуемые точки:
- В настройках `open_menu` (колбэк `_CB_MENU`) уже делает `state.clear()` + рендер меню —
  готовый «выход из мастера».
- В notify-флоу экран выбора момента рендерит `ask_when` (колбэк `_CB_NOTIFY_WHEN`),
  данные лежат в `state.get_data()[_NOTIFY_DATA_KEY]` отдельно от FSM-state.

## Goals / Non-Goals

**Goals:**
- На 7 шагах ввода настроек: кнопка «Отмена» (выход в меню без сохранения) + строка
  `Сейчас: {current}` под промптом.
- На шаге `notify_custom_time`: кнопка «Отмена».
- Минимум новых строк в `messages.toml`, единый формат показа текущего значения.

**Non-Goals:**
- Показ текущего значения на шаге `notify_custom_time` (это разовое время, не настройка).
- Кнопки выбора (таймзона, тоггл рабочих дней) — нет свободного ввода, не трогаем.
- Флоу клиента/записей/абонементов — «Отмена» там уже есть.
- Кнопка «Отмена» на шаге редактирования шаблона (`EditTemplate.body`) — отдельный шаг,
  вне заявленного скоупа; не трогаем в этом change.

## Decisions

### 1. Текущее значение — единый суффикс, не 7 переписанных промптов

`ask_value` грузит специалиста (`self._load`) и дописывает к промпту строку из новой
формат-строки `value_now = "\n\nСейчас: {current}"` (секция settings). 7 строк
`ask_*` остаются нетронутыми. Источник `{current}` — маппинг
`SettingField → getter(Specialist) -> str`, отдающий ровно то значение, что хранится
(`day_start`, `slot_minutes`, `subscription_presets` как есть, и т.д.).

*Альтернатива (отклонена):* `{current}` внутри каждой `ask_*` — естественнее звучит, но
переписывает 7 строк и плодит формат-параметры; пользователь принял «отдельной строкой».

### 2. «Отмена» в настройках = существующий `_CB_MENU`

Клавиатура шага ввода — одна кнопка с `callback_data=_CB_MENU`; обработчик `open_menu`
уже сбрасывает state и рендерит меню. Новый обработчик не нужен. Подпись — новая строка
`btn_cancel` в секции settings (а не reuse `btn_back = "⬅️ Назад"`: на шаге ввода
семантика именно «отменить ввод», не «назад по дереву»).

### 3. `notify_custom_time` «Отмена» = шаг назад к выбору момента

Кнопка «Отмена» на шаге ввода своего времени возвращает к экрану выбора момента
(`_notify_when_keyboard`), а не к карточке: пользователь на шаг раньше уже выбрал
«отправлять» — даём поменять способ (Сейчас / пресет / другое своё время), не теряя
notify-контекст. Реализация: новый колбэк-обработчик делает `state.set_state(None)`
(сбрасывает FSM-state ввода, но сохраняет `_NOTIFY_DATA_KEY` в data) и переиспользует
рендер `ask_when`.

*Альтернатива (отклонена):* отмена → карточка с «не отправлено» (как `notify_skip`).
Семантически это «не уведомлять вовсе», а пользователь хотел лишь выйти из ввода времени —
это потеря контекста на ровном месте.

### 4. Загрузка специалиста в `ask_value`

`ask_value` теперь делает один лишний `SELECT` специалиста ради `{current}`. Приемлемо:
шаг интерактивный (нажатие кнопки), не горячий путь; тот же `_load` уже используется
соседними хендлерами.

## Risks / Trade-offs

- [Рассинхрон промпта при гонке: специалист меняет поле из второго чата] → промпт
  показывает значение на момент входа в шаг; запись всё равно идёт через `update_setting`
  с валидацией, показанное «Сейчас» — справочное, не участвует в сохранении. Приемлемо.
- [Раздувание клавиатур/строк] → одна новая строка `btn_cancel` + одна `value_now`;
  кнопка «Отмена» в notify переиспользует существующий рендер. Минимально.

## Open Questions

Нет — формат показа значения и поведение «Отмены» в обоих местах зафиксированы выше.
