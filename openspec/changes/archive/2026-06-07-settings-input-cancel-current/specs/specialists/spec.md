## ADDED Requirements

### Requirement: Шаги ввода настроек предлагают отмену и показывают текущее значение

Бот **SHALL** на каждом шаге настроек со свободным текстовым вводом (начало рабочего
дня, конец рабочего дня, длина слота, время напоминаний, время утренней сводки, варианты
числа встреч в абонементе, время отложенной отправки) предлагать inline-кнопку «Отмена»
и показывать в промпте текущее значение поля. Нажатие «Отмена» **SHALL** выходить в меню
настроек без изменения значения и сбрасывать шаг ввода. Текущее значение носит справочный характер и не участвует в сохранении.
Шаги выбора готовыми кнопками (таймзона, тоггл рабочих дней) этим требованием не
затрагиваются — у них нет свободного ввода.

#### Scenario: Промпт показывает текущее значение

- **WHEN** специалист открывает шаг ввода одного из текстовых параметров настроек
- **THEN** бот показывает промпт с текущим сохранённым значением этого параметра
- **AND** показывает inline-кнопку «Отмена»

#### Scenario: Отмена выходит без сохранения

- **WHEN** специалист на шаге ввода нажимает «Отмена»
- **THEN** значение параметра не меняется
- **AND** бот возвращает экран меню настроек, шаг ввода сброшен

#### Scenario: Ввод нового значения сохраняется как прежде

- **WHEN** специалист на шаге ввода присылает корректное новое значение
- **THEN** значение сохраняется и применяется (поведение прежних требований не меняется)
