## Context

Раздел «Окна» (`src/bot/handlers/windows.py`) — reply-кнопка главного меню, по которой `WindowsHandlers.show` один раз отвечает списком всех свободных слотов на 5 рабочих дней. Расчёт — `list_free_windows` в `src/services/appointments.py`: для каждого дня берётся сетка `generate_slots(day_start, day_end, slot_minutes)`, из неё вычитаются занятые слоты (реальные записи + повторы активных серий), для сегодня отсекаются прошедшие.

Сетка слотов равномерная (шаг = `slot_minutes`), поэтому «соседство по времени» эквивалентно соседству по индексу в списке `slots` — это и есть основа режима «смежные».

Callback-переключение — устоявшийся в проекте паттерн: `router.callback_query.middleware(SpecialistMiddleware(...))` + `_callback_message(callback).edit_text(...)` с инъекцией `specialist_id` (см. `src/bot/handlers/clients.py`).

## Goals / Non-Goals

**Goals:**
- Два режима в разделе «Окна» — «Все окна» (текущее поведение) и «Смежные окна», переключаемые inline-кнопками в самом сообщении.
- «Смежное» окно — свободный слот, у которого соседний по сетке слот (в пределах того же дня) занят.
- Минимальный диф: развилка в одном месте расчёта + один новый callback-хендлер.

**Non-Goals:**
- Никакого виртуального «якоря» на границах дня (`day_start`/`day_end` не считаются занятыми соседями).
- Никакого показа на пустом дне в режиме «смежные».
- Не персистим выбранный режим в БД — состояние живёт в `callback_data`.
- Не трогаем фильтр прошедших окон и логику занятости (записи/серии).

## Decisions

### Решение 1: «Чистое» соседство, без якорей на границах дня

Свободный слот попадает в «смежные» только если **реальный** соседний слот по сетке занят:

```python
free = [
    s for i, s in enumerate(slots)
    if s not in taken
    and ((i and slots[i - 1] in taken) or (i + 1 < len(slots) and slots[i + 1] in taken))
]
```

Альтернатива — считать `day_start` виртуальным якорем, чтобы записи липли к утру и пустой день показывал первое окно. Отвергнута: усложняет правило и определение «смежного», порождает неочевидное поведение на пустом дне. YAGNI — если понадобится, добавим отдельным решением. Следствие принимается сознательно: пустой день в «смежных» = «свободных окон нет», для записи на пустой день специалист жмёт «Все окна».

**Тонкость, работающая бесплатно:** занятый слот в прошлом (сегодня) остаётся валидным якорем для будущего соседа — записи в 09:00 при `now=09:30` достаточно, чтобы 10:00 попало в «смежные». Соседство считается по полному `taken` (включая прошедшие занятые), а кандидаты в `free` уже отфильтрованы по будущему тем же шагом, что и сейчас. Порядок: сначала фильтр соседства по полной сетке, затем существующий фильтр прошедших слотов для сегодня.

### Решение 2: Режим как параметр сервиса + `callback_data`

`list_free_windows` получает параметр режима (булев флаг `adjacent` либо маленький `Enum` — выбрать при реализации, флаг проще и достаточен для двух режимов). Хендлер `show` рендерит дефолт «Все окна» с inline-клавиатурой из двух кнопок; новый callback-хендлер читает режим из `callback_data` (`windows:all` / `windows:adjacent`), пересобирает окна и `edit_text`. Активная кнопка помечается (например, `●`). Состояние полностью в `callback_data` — БД не нужна.

Альтернатива — хранить режим в FSM/настройках. Отвергнута: лишнее состояние ради двух кнопок.

### Решение 3: Рендер знает активный режим

`render_windows` получает активный режим, чтобы построить клавиатуру с пометкой и (опционально) корректным заголовком. Пустой день по-прежнему рендерится строкой `empty_day` — в «смежных» она будет частой, это ожидаемо и не требует отдельного текста.

## Risks / Trade-offs

- **Пустой день в «смежных» выглядит «всё занято», хотя день пуст** → Митигировано UX-ом: дефолт — «Все окна», переключатель всегда на виду; поведение задокументировано в спеке как сознательное.
- **Соседство опирается на равномерность сетки** → `generate_slots` гарантирует равномерный шаг; соседство по индексу корректно. Если сетка когда-нибудь станет неравномерной — правило придётся пересмотреть (зафиксировано здесь).
- **Дублирование загрузки данных в `show` и callback-хендлере** → Митигировано выделением общего приватного метода сборки окон по режиму.
