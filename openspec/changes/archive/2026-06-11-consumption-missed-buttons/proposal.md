## Why

В вечернем отчёте по расходу абонементов ❗-встречи (прошли, но не списано) показаны
только текстом и никуда не ведут. Специалисту, увидев «Ярослав Шилов (16:30) — активного
абонемента нет», приходится вручную искать клиента, чтобы разобраться. Списанные абонементы
рядом уже кнопки — поведение несимметрично.

## What Changes

- ❗-встречи в отчёте становятся **кнопками** (текстовые ❗-строки из сообщения убираются,
  причина переезжает в подпись кнопки).
- Причина «активного абонемента нет» (NO_SUBSCRIPTION) → кнопка ведёт на **карточку клиента**.
- Причина «абонемент закончился» (EXHAUSTED) → кнопка ведёт на **карточку абонемента**
  (он существует с `remaining = 0`, специалист может сразу продлить).
- Заголовки отчёта («Списано…», «Прошли, но не списано:») остаются текстом, чтобы сообщение
  читалось и без тапа.

### Modified Capabilities
- `subscription-consumption`: требование «Отчёт специалисту по итогам прохода» — ❗-встречи
  теперь не текстовые строки, а кнопки перехода (NO_SUBSCRIPTION → карточка клиента,
  EXHAUSTED → карточка абонемента).

## Impact

- `src/services/consumption.py` — `MissedEntry` обзаводится `client_id` и
  `subscription_id: int | None`; заполняются в `_record_outcome`.
- `src/bot/scheduler.py` — `_consumption_keyboard` отдаёт кнопки и для `report.missed`
  (callback по `reason`); `render_consumption_report` перестаёт печатать ❗-строки.
- `src/bot/messages.toml` — `missed_no_subscription` / `missed_exhausted` становятся
  подписями кнопок.
- `docs/features.md` / `docs/bot.md` — если отчёт описан, поправить.
- Тесты `tests/` по отчёту расхода — проверить кнопки для обеих причин и их callback.
- Без миграций БД, без новых зависимостей, без гонок. Хендлеры `clients:card:` и
  `subs:card:` уже существуют.
