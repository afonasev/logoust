## Context

Выбор времени для записи использует сетку слотов специалиста (`generate_slots`,
шаг `slot_minutes`). Занятость считает `taken_slot_times` (`services/appointments.py`):
она собирает множество настенных `HH:MM` **начал** записей за день — разовых из БД и
будущих повторов регулярных серий (`slot_taken_times` из `services/recurring.py`). Клавиатуры
`build_slots_keyboard` / `build_recur_slots_keyboard` подсвечивают слот через `slot in taken`.

Два дефекта:
1. Все регулярные пикеры слотов (`pick_weekday:2213`, `start_slot_time:2446`, и путь
   `start_slot_day`→`pick_weekday`) передают пустой `set()` — занятость не считается.
2. Матчинг `slot in taken` — точное равенство строк, поэтому запись вне сетки (14:10) не
   совпадает ни с одной ячейкой и не подсвечивается, а правило перекрытия не реализовано.

## Goals / Non-Goals

**Goals:**
- Единое правило занятости по пересечению интервалов во всех пикерах слотов.
- Включить подсветку в регулярных пикерах; учитывать разовые + регулярные записи.
- Регулярный выбор слота считает занятость по ближайшей дате дня недели от сегодня.
- При правке слота — отдельный маркер для его собственного текущего слота.
- Текстовый список записей дня перед сеткой во всех пикерах.

**Non-Goals:**
- Жёсткий запрет ставить встречу поверх занятого слота — подсветка остаётся
  предупреждением, выбор не блокируется (как сейчас).
- Поддержка разной длительности у записи и слота — длительность записи равна `slot_minutes`.
- Изменение раздела «Окна» (`availability`) — там своя логика свободных окон.

## Decisions

### 1. Правило занятости — пересечение полуоткрытых интервалов, в чистой функции

Слот `s` занят записью `a`, если `[a, a+slot) ∩ [s, s+slot) ≠ ∅`. При равной длительности
это `|a − s| < slot_minutes` (минуты). Касание краёв не считается (запись 14:00 не занимает
14:30 при 30-мин слоте).

Реализация — новая чистая функция в `domain/schedule.py` рядом с `generate_slots`:
`occupied_grid_slots(grid: list[str], starts: set[str], slot_minutes: int) -> set[str]`.
Она разворачивает множество **начал** записей в множество ячеек сетки, которые они
перекрывают. `taken_slot_times` продолжает возвращать начала; трансформация применяется в
хендлере перед построением клавиатуры.

_Альтернатива:_ считать перекрытие прямо в `build_*_keyboard`. Отклонено — клавиатуры
остаются «глупыми» (`slot in occupied`), а логика перекрытия живёт в одной тестируемой
чистой функции.

### 2. Регулярный пикер — опорная дата = ближайший день недели от сегодня

`taken_slot_times` работает по конкретной дате. В регулярных пикерах резолвим выбранный
`weekday` в `next_weekday_on_or_after(today, weekday)` — ту же дату, что `_append_slot:2271`
кладёт в `start_date` нового слота. Так подсветка соответствует реально создаваемому слоту.

### 3. Правка слота — исключение себя + третий маркер

При правке существующего слота (`flow ∈ {cfg_time, cfg_day}`) его собственная occurrence
не должна гореть «занято». Решение:
- Проброс `exclude_slot_id` через `taken_slot_times` → `slot_taken_times`: вклад этого слота
  в набор занятых не учитывается (по аналогии с `exclude_id` для разовой записи).
- Ячейка сетки, равная текущему `slot.time_hhmm`, помечается **третьим маркером**
  («текущее время») — клавиатура `build_recur_slots_keyboard` получает опциональный
  `current: str | None`. Маркер имеет приоритет над «занято»/«свободно».

В пикерах создания (`rcreate`, `cfg_add`) `exclude_slot_id`/`current` отсутствуют — обычные
🟢/🔴.

### 4. Текстовый список записей дня — переиспользование `list_specialist_day`

Перед сеткой во всех пикерах печатаем записи дня. `list_specialist_day(repo, …, day, tz,
series)` уже отдаёт слитый список occurrence (разовые + регулярные) за день; имена —
`client_name_map`. Новый чистый рендер `render_day_appointments_list(day, appts, names) ->
str` форматирует: заголовок-дата (с днём недели) + строки `HH:MM — Имя (комментарий)`.
Off-grid записи кнопки не имеют, но в тексте видны — это и есть подстраховка.

## Risks / Trade-offs

- **Запись вне сетки занимает две соседние ячейки, а кнопки на её точное время нет** →
  ожидаемо: точное время доступно через «Другое время», а текстовый список показывает
  реальное время записи. Документируем в сценарии.
- **`exclude_slot_id` исключает только будущий вклад правимого слота, не материализованную
  прошлую строку** → правка касается только будущих дат (`date >= today`), материализованное
  прошлое в пикер не попадает. Риск отсутствует.
- **Сравнение времён как настенных минут одного дня** → корректно: все начала и слоты — в
  таймзоне специалиста на одну дату, DST в пределах суток не разрывает сетку.
- **Третий маркер усложняет клавиатуру** → ограничен регулярными пикерами правки; для
  разовой записи `build_slots_keyboard` не меняет сигнатуру.
