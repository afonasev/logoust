## Why

При выборе времени для **регулярной** записи сетка слотов всегда показывает все слоты
свободными — занятость не считается вообще (передаётся пустой набор). А там, где
занятость считается (разовая запись, перенос), подсветка сравнивает время начала записи
со временем слота точь-в-точь, поэтому запись, начатая не по сетке (например 14:10 при
часовой сетке), не помечает ни одного слота. Специалист рискует поставить встречу поверх
существующей, не увидев конфликта.

## What Changes

- Слот считается занятым, если интервал записи `[a, a+slot)` **пересекается** с интервалом
  слота `[s, s+slot)` (полуоткрытые интервалы: касание краёв не считается). При равной
  длительности это `|a − s| < slot_minutes`. Запись 14:10 при 30-мин слоте занимает и 14:00,
  и 14:30; запись ровно в 14:00 — только 14:00.
- Подсветка занятости включается во **всех** точках выбора слота, где её сейчас нет:
  создание регулярного слота, правка времени слота, правка дня слота.
- Занятость учитывает и разовые, и регулярные записи (включая повторы серий и переносы
  отдельных дат) — как уже делает разовая запись.
- Для регулярного выбора слота занятость и список считаются по **ближайшей подходящей дате**
  выбранного дня недели от сегодня (та же дата, что станет `start_date` слота).
- При правке существующего слота его собственный текущий слот помечается **отдельным
  маркером** («текущее время»), а не «занят», и исключается из набора занятых другими.
- Перед кнопками выбора слота во всех пикерах печатается **текстовый список записей дня**
  (время + имя + комментарий) — подстраховка, видны и записи вне сетки.

## Capabilities

### New Capabilities

(нет — меняется поведение существующих)

### Modified Capabilities

- `appointments`: правило занятости слота меняется с точного совпадения времени на
  пересечение интервалов; перед сеткой слотов показывается текстовый список записей дня.
- `recurring-appointments`: при создании/правке регулярного слота сетка показывает
  занятость (разовые + регулярные) по ближайшей дате дня недели, с отдельным маркером для
  собственного слота при правке и текстовым списком записей дня перед сеткой.

- `src/services/appointments.py` — новый чистый хелпер занятости по пересечению; проброс
