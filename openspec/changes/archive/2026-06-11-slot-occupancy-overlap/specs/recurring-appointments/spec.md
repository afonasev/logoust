## MODIFIED Requirements

### Requirement: Создание регулярной записи

Бот **SHALL** позволять специалисту из карточки клиента создать регулярное расписание —
набор **слотов**, каждый из которых задаёт день недели и настенное время `time_hhmm`, плюс
один общий комментарий на всё расписание. Расписание **SHALL** допускать произвольное число
слотов, в том числе несколько слотов в один день недели. Создаётся одно
`recurring_schedule` с `specialist_id` текущего специалиста, `client_id` выбранного
клиента, `comment` (опционально), `active=true`; для каждого слота — `recurring_slot` с
`weekday` (0=Пн…6=Вс), `time_hhmm`, `active=true` и `start_date`, равной ближайшему такому
дню недели на сегодня или позже в таймзоне специалиста. Время каждого слота выбирается из
сетки слотов расписания специалиста либо вводится произвольно как `ЧЧ:ММ` (с теми же
правилами валидации, что у разовой записи). Сетка выбора времени слота **SHALL** показывать
занятость по тому же правилу пересечения интервалов, что и разовая запись, учитывая разовые
записи и повторы активных регулярных серий на **ближайшую дату выбранного дня недели ≥
сегодня**; перед сеткой **SHALL** выводиться текстовый список записей этой даты.

#### Scenario: Создание расписания с несколькими слотами

- **WHEN** специалист в карточке клиента создаёт регулярное расписание, добавляет слот «Пн 12:00», добавляет слот «Вт 14:00» и подтверждает без комментария
- **THEN** создаётся одно `recurring_schedule` с `comment IS NULL`, `active=true` и два `recurring_slot` (Пн 12:00 и Вт 14:00), у каждого `start_date` — ближайший такой день недели ≥ сегодня
- **AND** бот подтверждает создание

#### Scenario: Несколько слотов в один день недели

- **WHEN** специалист добавляет в одно расписание слоты «Пн 12:00» и «Пн 16:00»
- **THEN** расписание содержит оба слота, и обе occurrence показываются на каждый понедельник независимо

#### Scenario: Подсветка занятости при выборе времени слота

- **WHEN** специалист выбрал день недели для нового слота, и на ближайшую дату этого дня недели ≥ сегодня у него уже есть записи (разовые или повторы серий)
- **THEN** в сетке слоты, чьи интервалы пересекаются с этими записями, помечаются занятыми, а над сеткой выводится текстовый список записей этой даты
- **AND** пометка не запрещает выбор занятого слота

#### Scenario: Создание слота с произвольным временем и комментарием

- **WHEN** специалист добавляет слот, вводит корректное произвольное время `ЧЧ:ММ` и задаёт непустой комментарий расписания
- **THEN** слот создаётся с этим временем, расписание — с этим комментарием

#### Scenario: Некорректное произвольное время

- **WHEN** специалист вводит произвольное время в неверном формате или вне диапазона `00:00`–`23:59`
- **THEN** слот не создаётся
- **AND** бот сообщает об ошибке формата и просит ввести время заново

### Requirement: Редактирование серии

Бот **SHALL** позволять специалисту настроить расписание из его карточки: по каждому слоту
изменить время или день недели, удалить слот, а также добавить новый слот. Правка **SHALL**
применяться только к будущим повторам (`date >= today`); уже материализованные прошлые
записи **MUST NOT** изменяться. При смене дня недели слота система **SHALL** пересчитать его
`start_date` на ближайший новый день недели ≥ сегодня и поднять `materialized_through` до
сегодня. Смена только времени слота сетку дат не меняет. При выборе нового времени или дня
слота сетка **SHALL** показывать занятость (разовые записи и повторы серий) по тому же
правилу пересечения интервалов, что и разовая запись, по ближайшей дате соответствующего дня
недели; собственный правимый слот **MUST NOT** считаться занятым, а его текущее время в сетке
**SHALL** помечаться отдельным маркером; перед сеткой **SHALL** выводиться текстовый список
записей этой даты. Удаление слота **SHALL** ставить `recurring_slot.active=false`, сохраняя
его прошлые записи; удаление последнего активного слота **SHALL** гасить расписание
(`schedule.active=false`).

#### Scenario: Смена времени слота

- **WHEN** специалист в настройке расписания меняет время слота на новое корректное
- **THEN** будущие повторы этого слота показываются по новому времени, а прошлые записи слота в истории остаются с прежним временем

#### Scenario: Смена дня недели слота

- **WHEN** специалист меняет день недели слота
- **THEN** `start_date` слота пересчитывается на ближайший новый день недели ≥ сегодня, будущие повторы идут по новому дню, прошлые записи прежней сетки не меняются

#### Scenario: Подсветка при правке слота с маркером собственного времени

- **WHEN** специалист правит время существующего слота и видит сетку слотов целевого дня
- **THEN** занятыми помечаются слоты, пересекающиеся с чужими записями и повторами, текущее время правимого слота помечается отдельным маркером и не считается занятым, а над сеткой выводится список записей этой даты

#### Scenario: Добавление слота

- **WHEN** специалист в настройке расписания добавляет новый слот с днём недели и временем
- **THEN** расписание получает дополнительный активный слот, его occurrence появляются в расписании наряду с остальными

#### Scenario: Удаление слота

- **WHEN** специалист удаляет один из нескольких слотов расписания
- **THEN** слот становится `active=false`, его будущие occurrence исчезают, прошлые записи слота остаются в истории, остальные слоты не затрагиваются

#### Scenario: Удаление последнего слота гасит расписание

- **WHEN** специалист удаляет единственный оставшийся активный слот
- **THEN** расписание становится `active=false`, будущих occurrence не остаётся, прошлые записи сохраняются

### Requirement: Перенос отдельной даты серии

Бот **SHALL** позволять специалисту из карточки одной встречи перенести эту occurrence
`(slot, original_date)` на другое время. Перенос **SHALL** записываться как override слота с
`moved_to` (UTC). На исходную плановую дату occurrence **SHALL** показываться по новому
времени/дню приземления, а плановый слот этой даты **MUST NOT** считаться занятым;
остальные occurrence не затрагиваются. При выборе новой даты сетка слотов **SHALL**
показывать занятость по тому же правилу пересечения интервалов, что и разовая запись (разовые
записи и повторы серий), а перед сеткой **SHALL** выводиться текстовый список записей этой
даты.

#### Scenario: Перенос будущей occurrence

- **WHEN** специалист открывает карточку конкретной будущей встречи, выбирает «Перенести» и задаёт новые корректные дату и время
- **THEN** создаётся (или обновляется) override слота для этой `original_date` с `moved_to` в UTC
- **AND** occurrence показывается по новому времени, исходный слот свободен, остальные occurrence не меняются

#### Scenario: Подсветка занятости при выборе даты переноса

- **WHEN** специалист выбрал новую дату для переноса occurrence и видит сетку слотов
- **THEN** слоты, пересекающиеся с записями и повторами этой даты, помечаются занятыми, а над сеткой выводится текстовый список записей этой даты
